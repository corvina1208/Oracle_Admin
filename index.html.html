■ 12c 이후부터 변경된 혁신적인 oracle database 기능

컨테이너 DB 라는 개념이 생겼다.

ORACLE multitenant 아키텍쳐에서 Pluggable database (PDB) 를 생성해서 DB 생성을 손쉽게 하고 PDB 간의 데이터 이행을 굉장히 손쉽게 하는 기능이 생겼다.

※ ORACLE multitenant 아키텍쳐 장점 ?

1) 비용 절감 : 라이선스 비용, 저장공간, 메모리 사용량 절감 효과
2) 빠른 프로비져닝 복제가 가능 : PDB 를 생성하고 복제하는 작업이 매우 빠르고 간단
3) 향상된 성능 모니터링 : CDB 내의 모든 PDB 에 대한 성능 모니터링 및 분석을 하나의 인터페이스에서 수행할 수 있다 (EM 사용)

■ 실습

#1. PDB 가 뭐가 있는지 확인한다.

sys> select name from v$pdbs;

#2. file_name_convert 파라미터를 조회한다.

pdb_file_name_convert ← pdb 생성 시 데이터 파일 경로 변환 규칙을 설정한다.

#3. pluggable database 생성하기

sys>
create pluggable database jsjdb
admin user sys identified by oracle_4U;

--> 에러

ERROR at line 2:
ORA-65001: missing or invalid administrative user name

( pdb 관리자를 만들어야 한다고 에러가 난다. 때문에 아래와 같이 수행한다. )

sys>
create pluggable database jsjdb
admin user hradmin identified by tiger roles=(dba);

--> Pluggable database created.

sys> select name, open_mode, open_time from v$pdbs;

#4. pdb jsjdb 를 open 시키기

sys> alter pluggable database jsjdb open read write;

sys> select name, open_mode, open_time from v$pdbs;

#5. 이제부터 jsjdb를 사용하겠다 라고 지정하고 사용한다.

sys> show con_name

sys> alter session set container=jsjdb;

sys> show con_name

#6. 현재 가지고 있는 data file들을 확인하기

sys> select tablespace_name, file_name from dba_data_files;

#7. 리스너를 올리고 리스너의 상태 확인하기

[orcl:~]$ lsnrctl start

[orcl:~]$ lsnrctl status

#8. jsj2 라는 이름으로 pluggable db를 생성하기

sys>
create pluggable database jsjdb2
admin user hradmin2 identified by tiger roles=(dba);
